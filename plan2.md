# 招聘协同平台 V2 实施计划

## 范围
- 保留现有 `PLAN.md` 作为一期方案，本次 V2 能力单独落在本文件。
- 本次聚焦四件事：真实 `MySQL + MinIO` 本地运行、流程主链路修复、简历解析增强、辅助决策 Agent 落地。

## 实施顺序
1. 先补齐本地环境：`docker-compose.yml`、`.env.example`、`docs/local-docker-setup.md`、`application-local.yml`、`application-test.yml`。
2. 再实现后端 V2：解析结果结构化、辅助决策 job/API、流程推进规则调整、预览链路增强。
3. 再实现前端 V2：新增页回列表、统一预览逻辑、候选人详情页辅助决策弹窗、解析结果展示。
4. 最后补测试并完成回归验证。

## 验收标准
- 本地按文档启动 Docker 后，后端可连接 `MySQL + MinIO`，并自动创建简历桶。
- 新增候选人保存后回到列表，能立即看到新候选人数据。
- 点击“查看简历”可直接打开 PDF 预览，不再静默失败。
- 候选人流程可完成“新建/简历池 -> 分发部门 -> 部门反馈 -> 多轮面试 -> Offer -> 录用/淘汰”的主链路。
- 简历解析能输出结构化结果、解析摘要、问题清单，并支持重新解析。
- 候选人详情页可触发“辅助决策”，弹窗展示最新结果并可查看历史记录。
- `mvn test` 在 `test` profile 下通过。

## 回滚点
- 若真实 MySQL / MinIO 接入出现阻塞，可临时切回 `test` profile 验证业务逻辑，但不回退 V2 代码结构。
- 若外部 Agent 未接通，保留本地 mock dispatcher 自动产出结构化解析与辅助决策结果。
