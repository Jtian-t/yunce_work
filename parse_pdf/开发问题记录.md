# 开发问题记录

## 2026-04-22 火山引擎豆包 JSON 输出模式兼容性

### 当前环境

- 当前接入模型：`doubao-seed-1-8-251228`
- 当前接入方式：火山方舟 OpenAI 兼容接口
- 当前 Base URL：`https://ark.cn-beijing.volces.com/api/v3`
- 实际开发中建议优先使用方舟推理接入点 ID 作为 `model`，例如 `ep-xxxxxxxx`

### 问题现象

- 当请求中携带 `response_format={"type": "json_object"}` 时，如果 `messages` 中没有明确出现 `json` 这个词，接口会直接返回 `400 BadRequest`
- 典型报错：

```text
'messages' must contain the word 'json' in some form, to use 'response_format'
```

### 原因分析

- `response_format={"type": "json_object"}` 不只是提示模型“尽量输出 JSON”，而是开启了服务端的 JSON 模式校验
- 对当前豆包模型/接入点来说：
  - 如果开启了 `response_format={"type": "json_object"}`，提示词里必须明确写“JSON”或“json 格式返回”
  - 如果不想要求 JSON 输出，就不要传 `response_format`
- 即使不传 `response_format`，模型仍然可以在提示词约束下输出“像 JSON 的文本”；但这不等同于服务端原生 JSON 模式

### 开发结论

- 需要稳定拿到结构化结果时：
  - 优先尝试 `response_format={"type": "json_object"}`
  - 同时在 `system` 或 `user` prompt 中明确写出“请仅返回 JSON”
- 如果当前模型或接入点不支持或限制较多：
  - 降级为普通文本输出
  - 再从返回文本中提取 JSON

### 当前代码策略

- `src/llm_client.py` 已兼容两种模式：
  - 优先尝试 `response_format={"type": "json_object"}`
  - 如果当前模型/接入点不支持，则自动降级为普通文本输出
- 直连调试脚本：
  - `tests/test_llm_direct.py`
